Inline Labels - Use Cases

Simple Case

InlineLabel initialized with a CSS selector identifying the only field of the form. The form field has both a default value and a label associated with it, it gives precedency for the default value, therefore keeping the label as it is.

When the submit button is clicked before the form is posted the input:text if not changed gets its original value.

Multiple Selectors

InlineLabel initialized with both form fields (input:text and textarea). The input field doesn't have a default value, so it's label is used and removed (by default) from the form. The textarea has both a default value and a label, so the default value takes precedence and the label is kept as it is (not removed).

When the form is subimitted the input:text becomes empty and the textarea keeps its original default value.
